# 框架 Bug 审查报告(2026-07-12, 3-agent 对抗审查 + 交叉验证)

> 审查 LMWM×LaWM 对比实验的整条管线(P1 数据→训练→P2 adapter→P3 starVLA 集成)。
> **结论:发现 1 个 CRITICAL 设计缺陷——当前实验根本没在测"milestone vs next-frame"假设。** 另有 2 个 MAJOR + 1 个 MODERATE。数据管线本身正确。

---

## 🔴 CRITICAL-1:世界模型目标是 t+7 近未来帧,不是 milestone+1 → 对比无效

**证据链**(lawam.py / lerobot_datasets.py / train_libero.yaml):
1. `sec_chunk=0.4` × fps 20 = 8 步 → `video_delta_indices=[0,7]`(**不是** future_action_window_size=49 控制;那个只进 flow head)。
2. `h_t1_gt = lam.extract_vision_features(primary_video)[:,-1]` = **t+7 帧特征**;LMWM swap **只换 decoder、共享 extract_vision_features** → **baseline 与 LMWM arm 的 h_t1_gt 逐比特相同**。
3. `loss_perceptual = MSE(decoder(h_t,code), h_t1_gt=t+7)`;`unfreeze_lam_decoder=true` + `detach_future_feature=true` → **decoder 唯一梯度来自这个 MSE**。
4. → 我们训好的 MilestoneGenerator 被 25000 步 lr=1e-4 纯拉去拟合 t+7 帧,**milestone 预训练语义被覆盖,只剩初始化**。

**诚实判断:两个 arm 用同一个 next-frame 目标训练,唯一区别 = decoder 架构 + 初始化。milestone 假设从未进入监督信号。对比相对宣称的假设无效。**

**修法(必须):** 让 LMWM arm 的 `h_t1_gt` = **milestone+1 帧**的 DINO 特征,而非固定 delta 的 t+7 帧。= 改数据管线的视频采样为 **per-sample 动态**(当前帧 + 该帧的下一 milestone 帧)。这是本实验最大的一块工作(delta_indices 现为 dataset 级常量)。

---

## 🟠 MAJOR-2:code 空间失配(swap_teacher 默认 False)

`LMWM_SWAP_TEACHER` 默认 "0" → `get_latent_action` 仍是原 LaWM 量化器(LaWM code 空间),`enable_loss_distill=true` 把 `vlm_to_lam` 蒸馏去匹配 **LaWM code**,再喂进我们按 **LMWM InverseEnc code** 训练的 generator → FiLM 调制 OOD。decoder 解冻会重训 mod 去解释 LaWM code,**再次抹掉 LMWM 身份**。
**修法:** `LMWM_SWAP_TEACHER=1`(用 LMWM InverseEnc 产 code,自洽)+ `make_lmwm_lam` 加 `assert base_lam.code_dim==code_dim`(硬编码 32 无校验)。

## 🟠 MAJOR-3:baseline 必须同管线 no-swap,不能用 released LaWM 数字

唯一干净 baseline = **同一 starVLA config + 同 pretrain + 同 25000 步,只是不设 LMWM_CKPT**(原 decoder)。用 released LaWM 的 SR(不同 recipe/step/数据)= provenance 混淆,不可比。
**修法:** 对照组也自训(no-swap),不用 released 数字。

## 🟡 MODERATE-4:milestone readout 用硬 cummax(棘轮效应)

`p1_libero_milestone_pairs.py:56-58` `np.maximum.accumulate(argmin)` 是不可逆棘轮——单个早期噪声帧最近邻到高序质心,永久锁死该 episode 后续所有帧 ms≥k,**静默污染部分 episode 的训练对**。CRAVE 规范单调 readout 是 **Viterbi**(转移惩罚),不用硬 cummax。
**修法:** 改用 `crave.value.readout_viterbi_ms` 或加转移惩罚/中值平滑。

## 🟢 MINOR
- adapter 存了 `prd`(部署头)但 `make_lmwm_lam` 从不用——自主 rollout 路径(无未来帧)靠 VLA action expert 出 code,prd 未接线,需确认。
- `extract.py:49` 只读 episodes-meta 第一个 shard(现 1693 全在内,无 bug;分片后会静默漏)。
- sec_chunk=0.4→8 步 vs action_horizon=50/future=49 配置冗余,建议核对 flow 实际 action 步数。

## ✅ 经严格验证正确(无 bug)
- 视频 seek+stride+帧对齐(对 meta 时间戳逐一验证,开区间 [from,to) 恰好 length 帧,不漏不重)。
- grid reshape [N,256,768]↔[N,768,16,16] token 顺序:训练与 adapter **逐元素一致**(`R[·,d,i,j]=g[·,i*16+j,d]`)。
- gist pooling `mean(1)` = CRAVE `encode_pooled` 语义。
- per-task 分组、first[m+1] 无时间倒流(cummax 保证 first[m+1]>i)、cur_fi/tgt_fi 与 npz 对齐。
- MilestoneGenerator/InverseEnc 类定义训练↔adapter 一致,load_state_dict 安全;lift 方向对;LMWMDecoder 形状链自洽无 crash。
