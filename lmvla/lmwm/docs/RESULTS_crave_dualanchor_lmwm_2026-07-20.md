## 2. 先证否:CRAVE 原版「全局双锚」在 libero10 上是空操作

按 CRAVE 原文照搬(起点锚=全ep首3帧均值,终点锚=全ep末3帧均值):

| 指标 | 现状(单锚) | 全局双锚 |
|---|---|---|
| 段数 mean | 6.49 | 7.86 |
| **真尾巴**(最后一个非锚段) | **0.245** | **0.238** |
| 真尾巴 > 30% 的 ep 占比 | 0.309 | 0.307 |
| 终点锚吸走帧占比 | — | **0.004** |

> ⚠️ 若按「最终段占比」看会得到 0.245 → **0.004** 的假象性大胜 —— 那只是终点锚吃掉了被强制的**末帧本身**,
> 原来的大尾巴变成了倒数第二段。**必须用「最后一个非锚段」度量**,否则结论完全反了。

**为什么失效**:libero10 是 **10 个不同任务混合**,全局末帧均值是**跨任务糊平均**,不是任何一个任务的完成态。
实测任务间末锚 `cos` 仅 **0.864**(彼此差得远),而终点锚到最近 milestone 的距离 **0.267** —— 除被强制的末帧外没有帧愿意去。
CRAVE 在 kai0 上有效,是因为那是 **3055 ep 的单一折叠任务**,末帧均值确实 = 「cloth-average 完成态」。

**附带数据**:libero10 上 `cos(起点锚, 终点锚) = 0.9753` —— 首末别名在 LIBERO 同样严重,不是 kai0 独有。

## 4. 判据:同 recipe 重训,3 seed

两臂除标签外**完全相同**(proto teacher · shared PCA128 · code_dim 128 · K=4 · seglast · 9000 步 · lift_w 1.0)。

**共同口径**(与标签无关,可直接相减):每个 val ep 均匀采 12 帧 → 预测该 ep **真实末帧**,报 `deploy − persist`。
seed ∈ {2026, 7, 42}。

| 位置 | base | **per-task 双锚** | Δ | 3 seed 同向 |
|---|---|---|---|---|
| 0-10% | +0.0112±0.0005 | +0.0129±0.0024 | +0.0016 | 否 |
| 10-20% | +0.0364±0.0033 | +0.0331±0.0036 | **−0.0032** | 是 |
| 20-30% | +0.0478±0.0052 | +0.0461±0.0049 | **−0.0017** | 是 |
| 30-40% | +0.0496±0.0021 | +0.0507±0.0023 | +0.0011 | 是 |
| 40-50% | +0.0291±0.0021 | +0.0321±0.0033 | +0.0031 | 是 |
| 50-60% | +0.0184±0.0022 | +0.0249±0.0027 | +0.0065 | 是 |
| 60-70% | +0.0266±0.0026 | +0.0340±0.0022 | +0.0074 | 是 |
| 70-80% | +0.0305±0.0023 | +0.0371±0.0028 | +0.0066 | 是 |
| 80-90% | +0.0402±0.0035 | +0.0466±0.0007 | +0.0064 | 是 |
| **90-100%** | **−0.0292±0.0047** | **+0.0043±0.0019** | **+0.0335** | **是** |
| **全段** | **+0.0203±0.0002** | **+0.0283±0.0021** | **+0.0080** | **是** |

per-task Δlift(3 seed 均值):**10/10 为正**。
task 0 **+0.0305** / task 2 +0.0163 / task 8(白杯+布丁)**+0.0109** / task 9 +0.0074;
task 3、6 仅 +0.0004 / +0.0001 且 3 seed 未全同向 ≈ **无效果**(这两个任务本来尾巴就最小)。

**三点读数:**

1. **末端由负转正**。此前 [§3.5](RESULTS_p1_vs_final_terminal_2026-07-20.md#35-同口径闭合全帧对末帧两线可直接相减) 记录终版 90-100% 也是负(−0.0102),
   并据此判定「末端负 lift 是 `persist→1.0` 的结构性假象、不可当缺陷证据」。
   **本次修正该判断的适用范围**:它对**没有终端监督**的标签成立;一旦终点锚把完成态变成一个真实可达状态,
   末端**可以**做到非负。所以那是**可修的**,不只是度量假象。
2. **增益位置与机理吻合**。改善单调集中在 **50-100%**,正是 per-task 锚吸帧的区间;
   10-30% 有稳定但小的退化(−0.002~−0.003,3 seed 同向)—— 多出 20 个锚状态摊薄了早期段的监督,是真实代价,不是噪声。
3. **兑现了 §5 的机理要求**。终点锚给了 generator 真正的 (尾巴帧 → 完成态) 训练对,
   而非仅在 predictor 侧补标签 —— 这正是上一轮 `oracle ≈ deploy`(gap 0.001)所指出的必要条件。

> ⚠️ **不可用的对照**:两臂「各自 val pair」上的 lift(base 0.0486 vs anchor 0.0362)**不能比**——
> 标签不同则目标帧不同,anchor 臂的 persist 基线更高(0.9278 vs 0.9123)。只有 §4 的共同口径可比。

## 5. 未解决

- **milestone 粒度(第三缺陷)仍在**:task 0/9 尾巴 0.468/0.436 → 0.357/0.350,是全场最大,双锚只削掉一部分。仍需提高 per-task K。
- **只测了 libero10**。kai0(单任务,CRAVE 双锚的原生场景)未复测;RoboTwin 仍卡在 `task_index` 指令级分组(921k 变体)。
- **per-task 锚需要任务标签**。libero10 有 `meta/tasks.jsonl`;若数据集无任务划分,需先聚类出「任务」才能用,这是该方法的前提条件。
- 未测 CRAVE §2.12 的 **polyline 读出**是否也该进 LMWM(当前 medoid 已等价其锚点,但段间线性插值没做)。
